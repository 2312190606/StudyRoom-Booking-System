## 遇到的问题和解决
1. **问题**: 早期在编写 `architecture.md` 文档时，编写的 Mermaid 代码在部分 Markdown 预览工具（及 IDE 内置渲染器）中提示 “Parse error” 以及 “Lexical error”，造成图表破版无法展现；且后端结构网线过多导致图表混乱。
2. **解决**:  
   - 通过排查，发现是包含空格、特殊括号等混合文本未做转义引发解析错误。通过使用严谨的**双引号 `""` 包裹节点标签**，并且将简写的 `class` 设定替换为标准的跨兼容 **`style` 语法**，成功使图表恢复绘制。
   - 对冗杂的后端图表采取“抽象分层”手段，将具体的微逻辑（如多个 Controller 和 Service 节点的交叉）抽象为“逻辑分层（Layer）”的流向链路，大幅提升了图表的观感与结构条理。

在这次架构设计过程中，我不仅深入理解了自习室系统中的痛点需求，并以此针对性地作出了防超发抢票架构（Redis锁机制）和移动端轻快化等技术规划。更重要的是，在撰写标准架构图表的过程中，我掌握了 Mermaid 图表结构化的技巧：懂得了越复杂的系统流转，越需要以“高内聚低耦合”的 Layer 分层视角去做抽象表达，这样不仅代码维护成本低，更能让后续参与开发的其他同学一目了然。
