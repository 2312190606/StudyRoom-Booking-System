## 我完成的工作

### ### AI 安全审查
- **审查了哪些文件/模块**：
  - `AuthController.java` / `LoginAttemptService.java`（认证与锁定模块）
  - `AIController.java`（AI 聊天模块）
  - `AdminService.java`（管理后台业务模块）
  - `application.yml`（系统配置文件）
- **AI 发现的主要问题**：
  1. **登录暴力破解风险**：原代码缺乏有效的登录失败尝试限制，容易遭受暴力破解攻击。
  2. **配置循环引用漏洞**：`application.yml` 中 `DEEPSEEK_API_KEY` 存在循环占位符引用，导致系统在 Docker 环境下启动崩溃。
  3. **AI 提示词注入风险**：AI 聊天接口缺乏输入净化，攻击者可通过特定指令绕过系统设定。
  4. **管理员账号安全隐患**：管理员账号可能被误触发锁定或在后台被误禁用，导致运维通道中断。
- **我修复了哪些问题**：
  1. **实现登录锁定逻辑**：同一账号连续输错 5 次密码后，自动锁定 10 分钟。
  2. **修复配置逻辑**：重构了 `application.yml` 的 Key 结构，解决了启动崩溃问题。
  3. **加固 AI 安全**：增加了输入长度限制、HTML 净化以及基于关键词的注入指令拦截。
  4. **特权账号保护**：在代码层面确保管理员账号永不锁定、永不被禁用。

### ### 安全检查清单
- [x] **密码存储**：使用 BCrypt 强哈希加密存储。
