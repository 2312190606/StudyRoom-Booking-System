# 设计说明

## 配色方案
- 主色：#4F46E5（蓝紫色 - 应用于品牌色、主视觉背景、主要操作按钮等）
- 辅助色：#F8FAFC（浅灰色 - 大面积前台背景色）、#0F172A（深石板灰 - 管理后台深色背景）、#1E293B（深色内容卡片背景）
- 强调色：#10B981（成功绿 - 正常状态/已签到/生效中）、#EF4444（危险红 - 过期状态/封禁/删除操作）、#F59E0B（活力黄 - 评分与勋章）

## 字体
- 标题：PingFang SC / Microsoft YaHei（加粗用于模块标题及强调字面）
- 正文：PingFang SC / Inter（字母与数字优先使用 Inter 字体保障清晰度）

## 核心页面
1. **前台 - 首页**：展示各区域自习室的图文概览、座位剩余情况，提供全局通知提醒横幅及“一键预约”快捷入口。
2. **前台 - 我的预约**：查看当前生效的座位订单并生成签到二维码以便门禁扫码核销，同时支持查看历史预约记录。
3. **前台 - 个人中心**：结构化展示用户学习时长、天数等统计数据面板，并提供资料修改、密码重置等系统账户设置。
4. **后台 - 仪表盘**：全景数据监控，提供总预约量、活跃用户等核心指标看板，通过折线图展示流量趋势以及最新动态反馈。
5. **后台 - 自习室管理**：卡片式管理各区域自习室列表，支持维护可用座位数、配置空间布局、评价分数，以及快捷新增自习室。
6. **后台 - 预约记录/用户管理/公告管理**：功能集成的表格流管理系统，支持根据状态分类标签进行一键检索，包含封禁操作、导出报表功能。
7. **后台 - 系统设置**：配置系统级业务规则（如最大时长、签到窗口期），管理自动化防御策略（如检测占座违约），显示服务器运行监控信息。

## 交互说明
- **页面切换交互**：点击前台顶栏或后台侧边栏，高亮当前活跃菜单并平滑切换页面对应内容。
- **模块状态交互**：鼠标悬停（Hover）在自习室卡片或深色卡片列表时，出现微阴影浮动或背景高亮提升点击感知。
- **条件检索过滤**：后台管理列表顶部均支持输入关键字进行动态聚合检索，点击状态标签可直接切换展示目标数据分类。
- **弹窗与验证管理**：在点击“出示签到码”、“编辑”、“新增”等非常规流操作时触发页面居中弹窗/侧边抽屉进行聚焦操作。
- **快速开关控制**：在系统设置调整各类检测和状态权限时，开关（Switch）控件切换即触发接口变更、呈现即时开闭状态反馈。

## 用户研究

### 1. 明确目标用户群体
- **在校大学生/考研人群/职场考证者**：面临高强度独立学习压力，对图书馆或指定自习室座位有高频刚需，希望随时通过手机获知座位空闲情况，避免跑空。
- **自习室/图书馆管理员**：需要高效分配和调度座位资源，准确掌握实时入座率，清理并处理“占座不来”的违规行为，改善运营效率。

### 2. 核心用户场景
- **场景一：提前规划与预约选座**
  复习周即将来临，用户小明计划明天早上 8 点去图书馆自习。为了避免找不到座位，他提前一天晚上在宿舍通过手机打开系统，直观查看各区域的空闲座位分布图，并挑选了一个靠窗位置，提交并获得了明日早上的座位预约凭证。
- **场景二：现场入座与中途暂离**
  用户小李到达自习室后，扫描桌面上的座位二维码完成“入座签到”。中午到了就餐时间，她并不打算带走书本，于是在手机端点击“暂离”按钮，系统为她保留该座位 60 分钟，避免了因长期离开被视作违规而导致座位被系统回收。
- **场景三：管理员的高效纠察与释放利用**
  临近考试，座位非常紧张。管理员老王巡查时发现某个座位放着书包但很久无人，他通过管理员后台查验该座位状态，确认用户长时间离开且未点“暂离”，便直接在系统中标记违规并释放该座位。空出的座位立刻在前端显示为“可预约”，其他寻找座位的学生得以重新预订就座。
