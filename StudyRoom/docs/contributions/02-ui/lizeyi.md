### 页面设计

本系统涵盖移动端（前台用户层）与 PC 端（管理后台层），我负责了以下核心页面的高保真原型设计与交互流构建：

#### 移动端（前台部分）
- [x] **首页 (Home)**：构建了自习室发现卡片流，直观展示各大区域/自习室分类入口、全校实时余座概览及区域公告。顶部固定全局通知提醒横幅，并提供显著的“一键预订”快捷通道。
- [x] **选座与预订 (Booking)**：设计了交互式的可视化座位分布图，通过色彩区分座位状态（可选、已选、维修中等状态）。包含了多时间段滑动选择组件及实时的冲突校验面板，并在底部结合悬浮动作条完成规则提示与最终确认。
- [x] **行程状态 / 我的预约 (My Schedule)**：采用状态驱动的预约卡片流设计。针对当前生效订单，提供大尺寸签到二维码生成及入座签到按钮；提供“暂离 / 返回入座 / 提早签退”流控面板；并清晰展示历史预约记录。
- [x] **个人中心 (Profile)**：设计了用户专属的数据驾驶舱，呈现学习时长与天数统计表。集成了信用积分体系视图及违纪预警记录，并提供资料修改、密码重置等系统账户设置。

#### 管理后台（PC 端部分）
- [x] **仪表盘看板 (Dashboard)**：打造全景数据监控中心，包含今日座位利用率、活跃入座总人数折线图，以及违规占座处理预警等核心指标汇总看板。
- [x] **空间与区域管理**：采用卡片式布局结合详尽表单，支持维护自习室与所属分区设定、配置座位属性（总数、禁用位），以及可视化的平面图拓扑调整。
- [x] **综合动态管理体系（订单/用户/公告）**：采用功能集成的动态表格系统，支持顶部关键字输入动态聚合检索。包含全局预约单核销/手动释座、用户封禁与异常申诉处理，以及系统级公告横幅的发布与编辑。
- [x] **系统运行配置**：设计了配置系统级业务规则（如提前预约天数限制、违规上限、暂离窗口期等自动化防御策略）及呈现服务器监控状态的操作界面。

## 遇到的问题和解决
1. **问题**：在小屏幕下，座次表的网格图标容易产生误触且信息密度过高。
   **解决**：将座位图标设计得更具辨识度（颜色区分状态），并预留了足够的点击热区，同时通过增加靠窗标识减少用户的盲目寻找时间。
2. **问题**：多种预约状态（待签到、学习中、完成、取消）在视觉上容易混淆。
   **解决**：通过色彩心理学进行状态映射（主色用于当前活跃、灰色用于历史、红色用于预警），并在预约卡片上增加明确的状态徽章。
